Анализ операционных показателей маркетплейса Olist (Brazil)
В данном проекте реализован полный цикл обработки данных (ETL) для анализа работы бразильского маркетплейса Olist. Проект охватывает этапы от автоматизированной загрузки сырых данных в базу до построения интерактивного бизнес-дашборда.

Интерактивный дашборд
[Посмотреть дашборд в Tableau Public](https://public.tableau.com/app/profile/danil.ten/viz/Book2_17773861966560/Dashboard1?publish=yes)

Стек технологий
Python (Pandas, SQLAlchemy) — автоматизация загрузки CSV-файлов в реляционную базу данных.
PostgreSQL — хранение данных и создание аналитических витрин.
SQL — написание сложных запросов (Joins, Window Functions, расчет разницы дат).
Tableau — визуализация ключевых бизнес-метрик.

Этапы реализации
1. Extraction & Loading (Python)
Разработан скрипт `main.py`, который:
- Автоматически создает соединение с PostgreSQL.
- Обрабатывает типы данных (особенно временные метки).
- Загружает 9 связанных таблиц в базу данных.

2. Transformation (SQL)
Написан SQL-запрос (`create_view.sql`) для формирования плоской витрины данных. Рассчитаны такие показатели, как:
- Фактическое время доставки заказа.
- Разница между ожидаемой и реальной датой доставки.
- Полная стоимость заказа (цена + фрахт).

3. Visualization (Tableau)
Создан интерактивный дашборд, позволяющий анализировать:
- Продажи: Динамика выручки по месяцам.
- Географию: Распределение заказов по штатам Бразилии (лидер — Сан-Паулу).
- Продуктовый портфель: Самые прибыльные категории товаров (лидеры — Health & Beauty, Watches & Gifts).

Как запустить проект
1. Скачайте датасет Olist с [Kaggle](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce).
2. Настройте подключение к вашей PostgreSQL в файле `main.py`.
3. Запустите `main.py` для загрузки данных.
4. Выполните запрос из `create_view.sql` для создания аналитической витрины.
